Loop 节点提前中断通常有以下几种原因：

  1. breakCondition 满足：检查是否设置了中断条件，且在第三次迭代时条件被触发
  2. maxIterations 限制：确认 maxIterations 是否设置为 3
  3. itemsPath 数组长度：如果数据源只有 3 个元素，迭代会自然结束
  4. 下游节点报错：如果 Loop 内部有 agent/code 节点执行失败，且 onError 策略为 stop

建议检查 Loop 节点的配置面板，重点查看 breakCondition 和 maxIterations 字段。
