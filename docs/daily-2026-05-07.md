# 2026-05-07 开发总结

## 一、AI 服务商接入

### SiliconFlow 硅基流动
- 连接、获取模型列表、聊天测试全部跑通
- Base URL：`https://api.siliconflow.cn`，程序自动补 `/v1`
- 默认模型：`deepseek-ai/DeepSeek-V3.2`

### DeepSeek
- 新增为独立服务商类型
- Base URL：`https://api.deepseek.com`，自动补 `/v1`
- 默认模型：`deepseek-v4-flash`（主）、`deepseek-v4-pro`

### API 格式支持
- `ModelProvider` 新增 `apiFormat` 字段，三种可选：
  - **OpenAI Chat Completions** — `/v1/chat/completions`（通用）
  - **OpenAI Responses** — `/v1/responses`（仅 OpenAI 官方）
  - **Anthropic Messages** — `/v1/messages`
- `api.ts` 新增 `ensureV1()` 函数，用户只需填域名，程序自动拼 `/v1` + 端点路径

### 模型连通性测试
- **单个模型测试**：每个模型行新增 ▶ 按钮，成功 ✓ / 失败 ⚠+错误详情（点击弹窗看完整日志）
- **全部模型测试**：Models 标题右侧「测试」按钮 → 费用提示弹窗 → 逐模型串行测试 → 结果汇总
- 错误信息精确到 HTTP 状态码（401/403/404/429/500 等）

## 二、服务商管理 UI 重构

### 添加服务商
- 移除弹出对话框，改为行内添加：点击 + 直接新增到列表，右侧即时编辑

### 模型分组
- `ProviderModel` 新增 `group` 字段
- 分组名支持**点击修改**（回车保存，Esc 取消），未设置则自动从模型 ID 前缀提取

### API Key 显示
- 密码框右侧新增 👁 按钮，按住显示明文，松开恢复隐藏

### 获取模型列表
- 弹窗内始终显示「重新获取」按钮，可根据当前 URL + Key 重新拉取

### 其他
- 移除服务商类型下拉框
- 默认所有服务商不启用（`isEnabled: false`）
- 深色模式下拉选项对比度修复（`focus:bg-primary/15`）

## 三、SettingsView 组件化重构

```
SettingsView.tsx (1100+ 行) → 拆分为 4 个文件：
├── SettingsView.tsx            ← 70 行壳（标签栏 + 路由）
├── SettingsGeneralTab.tsx      ← 90 行（语言、远程访问）
├── SettingsAppearanceTab.tsx   ← 50 行（主题切换）
└── SettingsModelsTab.tsx       ← 500 行（服务商 CRUD + 模型测试 + 3 个对话框）
```

**解决的问题**：对话框与数据不在同一作用域导致切换页签黑屏。

## 四、配置持久化

| 模式 | 存储位置 | 方式 |
|------|----------|------|
| Tauri 桌面 | `config/config.json` | Rust 统一单文件，按 key 读写 |
| 浏览器 dev | `localStorage.simper_config` | JSON，结构与 Tauri 端一致 |

- 3 个分散文件（`model.json` / `agents.json` / `workflow.json`）合并为 1 个
- `config/` 已加入 `.gitignore`，API Key 不会提交到仓库

## 五、Bug 修复

| 问题 | 根因 | 修复 |
|------|------|------|
| 选中工作流后拓扑不切换 | `handleWorkflowSelect` 只调了 `setActiveWorkflow`，未切换会话 | 改用 `openWorkflowSession` |
| 浏览器模式新建会话无效 | `createSession` 的 `set()` 在 try 块内，invoke 异常后跳过 | `set()` 移出 try |
| 删除服务商黑屏 | 对话框挂在 models 条件块外部 | 移入条件块内（重构后根除） |
| 服务商测试 404 / 无响应 | `@ai-sdk/openai` v3 默认走 `/responses` 而非 `/chat/completions` | 改用 `openai.chat()` |
